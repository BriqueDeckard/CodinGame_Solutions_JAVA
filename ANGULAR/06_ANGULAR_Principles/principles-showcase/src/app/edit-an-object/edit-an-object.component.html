2. Edit-an-object:

Users should be able to edit the object values in an < input > textbox. The textbox should both display the object's properties and update that properties as the user types.

That means data flows from the component class out to the screen and from the screen back to the class.

To automate that data flow, you can setup a two-way data binding between the < input > form element and the hero.name property.

So

< input id="stringValue" [ ( ngModel )] = "someObject.stringValue" placeholder="stringValue" >

gives :



{ { someObject.stringValue } } gives: {{someObject.stringValue}}
